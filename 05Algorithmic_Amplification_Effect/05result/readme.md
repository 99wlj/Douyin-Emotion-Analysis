
# 阶段 5 操作说明：算法放大效应分析 (Algorithmic Amplification Analysis)

## 1. 分析原理与设计逻辑 (Principles & Design)

本阶段旨在通过统计学方法，验证抖音推荐算法和用户互动行为是否存在对**高情绪化内容**的放大效应。

### 1.1 核心假设 (Hypothesis)
* **H1（情绪极化假设）**：视频的情绪越强烈（无论是极度积极还是极度消极），获得的互动量越高。即互动量与情绪得分呈 **U型（非线性）关系**，而非简单的线性关系。
* **H2（算法偏好假设）**：机器学习模型（模拟推荐算法）预测的“爆款概率”，也会受到情绪变量的显著影响。

### 1.2 变量构建 (Variable Construction)
为了验证上述假设，我们在原始数据基础上构建了以下衍生变量：

1.  **因变量 (Dependent Variables, Y)**：
    * **对数互动量 (`log_interactions`)**：
        * *定义*：$ln(\text{点赞} + \text{分享} + 1)$
        * *原理*：社交媒体的互动量通常服从**长尾分布**（幂律分布），极少数视频拥有极大流量。直接回归会导致模型偏差，取对数（Log-transformation）使其更接近正态分布，符合 OLS 回归的基本假设。
    * **算法预测分 (`predicted_score`)**：
        * *定义*：阶段 3 训练的随机森林模型输出的“高互动概率值”（0~1）。
        * *原理*：作为“推荐算法”的代理变量，用于检测算法机制本身是否倾向于推荐情绪化内容。

2.  **自变量 (Independent Variables, X)**：
    * **情绪强度 (`sentiment_intensity`)**：
        * *定义*：$| \text{sentiment\_score} - 0.5 |$
        * *原理*：将 0~1 的情绪分转换为“中性（0）到极端（0.5）”的强度指标。如果该变量系数显著为正，则证明了**“越极端越火”**。
    * **文本长度 (`text_length`)**：
        * *原理*：作为**控制变量 (Control Variable)**。排除因为“文案长短”不同而导致的互动差异，确保我们观察到的是纯粹的“情绪”影响。

### 1.3 模型选择 (Model)
采用 **普通最小二乘法 (OLS) 多元线性回归模型**：
$$Log(\text{Interaction}) = \beta_0 + \beta_1 \cdot \text{Sentiment\_Intensity} + \beta_2 \cdot \text{Text\_Length} + \epsilon$$

## 2. 结果分析 (Results Interpretation)

基于您生成的回归结果文件，详细解读如下：

### 2.1 真实传播放大效应 (Regression_Real_Amplification.csv)
*这是最核心的证据，反映了真实的用户互动反馈。*

| 变量                    | 系数 (Coef.) | 标准误 (Std.Err.) | P值 (P>\|t\|) | 结论           |
| :---------------------- | :----------- | :---------------- | :------------ | :------------- |
| **Intercept** (截距)    | 9.2488       | 0.007             | 0.000         | -              |
| **sentiment_intensity** | **0.1652**   | 0.024             | **0.000**     | **显著正相关** |
| **text_length**         | -0.0257      | 0.000             | 0.000         | 显著负相关     |

* **解读**：
    * `sentiment_intensity` 的系数为 **0.165**，且 **P < 0.01**。这意味着在控制了文案长度后，**情绪强度每增加 1 个单位，视频的互动量也会显著上升**。
    * **实证结论**：数据有力地支持了“情绪极化”假设。抖音用户倾向于对情绪强烈的视频进行转评赞，中性平庸的内容受到冷落。

### 2.2 算法偏好机制 (Regression_Algorithm_Preference.csv)
*这是对算法机制的侧面验证。*

| 变量                | 系数 (Coef.) | P值 (P>\|t\|) | 结论           |
| :------------------ | :----------- | :------------ | :------------- |
| **sentiment_score** | **-0.0622**  | 0.000         | **显著负相关** |

* **解读**：
    * `sentiment_score`（0代表消极，1代表积极）与预测分呈负相关。
    * 这暗示了在您的模型看来，**消极/负面情绪的内容（分数较低）比纯粹的积极内容（分数较高）有略高的概率成为爆款**。这符合“坏事传千里”的传播学规律，也侧面印证了算法对负面高唤醒情绪的捕捉。

## 3. 数据字典 (Data Dictionary)

本操作生成了两个关键的数据集，其字段定义如下：

### 3.1 完整过程数据表 (`stage5_full_data_processed.csv`)
包含所有原始特征及本阶段计算的新特征，每行代表一个视频。

| 字段名                | 数据类型 | 含义说明             | 来源/计算方式                                              |
| :-------------------- | :------- | :------------------- | :--------------------------------------------------------- |
| `url`                 | String   | 视频链接ID           | 原始数据                                                   |
| `text`                | String   | 视频文案内容         | 原始数据                                                   |
| `sentiment_score`     | Float    | **原始情绪分** (0~1) | NLP模型输出 (0=消极, 1=积极)                               |
| `sentiment_intensity` | Float    | **情绪强度** (0~0.5) | 计算：`abs(sentiment_score - 0.5)`。数值越大，情绪越极端。 |
| `total_interactions`  | Int      | **总互动量**         | 计算：`点赞量 + 分享量`                                    |
| `log_interactions`    | Float    | **对数互动量**       | 计算：`ln(total_interactions + 1)`。用于回归分析的因变量。 |
| `predicted_score`     | Float    | **算法预测分**       | 阶段3机器学习模型的预测输出 (0~1)。                        |
| `text_length`         | Int      | 文案字数             | 计算：`len(text)`。控制变量。                              |
| `emotion_label`       | String   | 情绪标签             | 如 "积极", "消极"。                                        |

### 3.2 回归结果统计表 (`Regression_...csv`)
用于直接展示统计显著性的表格。

| 字段名          | 含义         | 如何用于论文                                                                 |
| :-------------- | :----------- | :--------------------------------------------------------------------------- |
| `Unnamed: 0`    | **变量名称** | 指示该行数据属于哪个自变量（如 `sentiment_intensity`）。                     |
| `Coef.`         | **回归系数** | **最重要的指标**。正值代表促进作用，负值代表抑制作用。数值大小代表影响程度。 |
| `Std.Err.`      | 标准误       | 估计的误差范围。                                                             |
| `t`             | t统计量      | 检验系数是否为0的统计量。                                                    |
| `P>             | t            | `                                                                            | **P值 (显著性)** | 判断结果是否可信。**必须小于 0.05** 才能宣称“显著影响”。 |
| `[0.025 0.975]` | 95%置信区间  | 系数的可能波动范围。                                                         |

### 总结
通过**构建强度变量**和**对数回归模型**，成功地从统计学层面量化了“情绪”对“流量”的贡献。结果显示，**情绪强度是驱动互动量的关键因素（P<0.01）**，这为解释“算法为何偏爱标题党和情绪化内容”提供了坚实的实证依据。
